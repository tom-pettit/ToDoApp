People

Up 
post /people/editTask
Edits an existing task in the people data structure (editTask)

Query parameters

newTaskName (required)
Query Parameter — The new name of the task that is being edited
newDueDate (required)
Query Parameter — The new due date of the task that is being edited
newDesc (required)
Query Parameter — The new description of the task that is being edited
newPerson (required)
Query Parameter — The new name of the person to which the task that is being edited will belong

Produces

This API call produces the following media types according to the Accept request header; the media type will be conveyed by the Content-Type response header.
  • application/json

get /people/searchForTask/{taskNameQuery}/{dueDateQuery}/{personQuery}
Returns all the details for a specific tasks (viewTask)
One, two, or three of these parameters can be supplied by the user. The client-side JS deals with the handling of empty inputs and if the user leaves a field empty, then the request will be sent the value 'NULL' for that particular input field.

Query parameters

taskNameQuery (required)
Query Parameter — The name of the task being searched for
dueDateQuery (required)
Query Parameter — The due date of the task being searched for
personQuery (required)
Query Parameter — The name of the person being searched for

Produces

This API call produces the following media types according to the Accept request header; the media type will be conveyed by the Content-Type response header.
  • application/json

Up 
get /tasks/seeExtraTasks/{specificDate}
Returns all the tasks for the date specified in the request (seeExtraTasks)
On the home page, a maximum of 3 tasks can be shown for each day, and when the number of tasks is greater than 3 for a particular day, a button is displayed and when clicked, this GET request is called in order for all the tasks on that date to be shown.

Query parameters

specificDate (required)
Query Parameter — The date for which the extra tasks are due on.

Produces

This API call produces the following media types according to the Accept request header; the media type will be conveyed by the Content-Type response header.
  • application/json
